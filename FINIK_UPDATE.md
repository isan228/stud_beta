# Обновление: Использование официального пакета @mancho.devs/authorizer

## Изменения

Интеграция обновлена для использования официального пакета `@mancho.devs/authorizer` вместо собственной реализации.

### Преимущества

1. ✅ **Официальная поддержка** - пакет поддерживается командой Finik
2. ✅ **Протестирован** - используется в продакшене
3. ✅ **Консистентность** - одинаковый алгоритм для создания и валидации подписи
4. ✅ **Меньше кода** - не нужно поддерживать собственную реализацию

### Что изменилось

1. **Добавлен пакет** `@mancho.devs/authorizer` в `package.json`
2. **Обновлен** `utils/finikClient.js` - использует `Signer` из пакета
3. **Обновлен** `utils/finikValidator.js` - использует `Signer` для построения канонической строки
4. **Файл** `utils/finikSigner.js` больше не используется (можно удалить, но оставлен для справки)

### Установка

```bash
npm install
```

Это установит `@mancho.devs/authorizer` автоматически.

### Использование

Код работает точно так же, как и раньше. Все изменения внутренние:

```javascript
// Теперь используется официальный пакет
const { Signer } = require('@mancho.devs/authorizer');

const signer = new Signer(requestData);
const signature = await signer.sign(privateKeyPem);
```

### Обратная совместимость

✅ Все существующие функции работают без изменений
✅ API endpoints остались прежними
✅ Формат конфигурации не изменился

### Миграция

Никаких действий не требуется! Просто установите зависимости:

```bash
npm install
```

И перезапустите сервер.
